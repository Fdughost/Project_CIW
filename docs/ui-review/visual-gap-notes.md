# Chloe in Wonderland Visual Gap Notes

本文件记录当前参考图与实际网页之间的主要差距，用于后续 UI 迭代自检。

## 2026-05-30 视觉闭环第一轮

参考目标是 `docs/design/gpt-reference/storybook-archive-ch*-desktop.png`：左侧为打开的私人童话档案书，右侧为旧木桌上的真实道具，顶部 HUD 像游戏内饰件，而不是网页导航。

### 已确认的主要差距

- 原实现有明显网页仪表盘感：顶部栏、右侧卡片、底部提示栏是分离的页面区块，而参考图更像一个完整桌面场景。
- 原底部提示栏导致桌面端页面过高，破坏参考图的单屏故事书构图。
- 右侧道具虽然已有交互，但仍偏表单和面板，缺少“物件躺在桌面上”的透视与阴影关系。
- 顶部道具栏塞入 8 个收集物后容易与右侧操作按钮挤压，和参考图中较克制的道具槽密度不同。
- 直接裁剪完整渲染图作为背景会产生两套 UI 互相打架：渲染图里已有假文字、假按钮、假透视，真实 DOM 叠上去会显脏。

### 本轮架构修正

- 桌面端改为固定单屏工作台：顶部 HUD + 左书页 + 右道具桌，不再依赖页面底部大栏。
- 提示系统移入故事书页脚，作为书内提示抽屉，而不是外部网页 footer。
- 右侧 puzzle stage 去掉厚重外框，改为透明道具承载层，让每章道具自己形成视觉主体。
- 顶部道具栏缩小密度并隐藏滚动条，避免后几章道具溢出。
- 移动端保留纵向堆叠，避免桌面单屏策略损害手机可操作性。

### 后续应继续解决

- 第一章需要把口琴、乐谱、木门锁进一步做成独立道具，而不是仍像分组卡片。
- 第二章守卫卡需要增加角色剪影、徽章和假设板的材质层次。
- 第四章 Nokia 的区号纸条和手机应形成更真实的桌面压影。
- 第八章需要增加月光窗口、冷光束和卡片背面小预览，减少普通输入控件感。
- 若继续使用 AI 图像，必须生成干净场景底图和透明道具 sprite，不能使用完整 UI 截图当背景。
